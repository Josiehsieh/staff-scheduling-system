# 🌐 如何開啟您的 GitHub Pages 網站

## 📍 您的網站網址

```
https://josiehsieh.github.io/staff-scheduling-system/
```

---

## 🚀 步驟 1：啟用 GitHub Pages（必須先做！）

### 1.1 前往您的 Repository

在瀏覽器中開啟：
```
https://github.com/Josiehsieh/staff-scheduling-system
```

### 1.2 進入 Settings

- 點擊頂部選單的「**Settings**」（齒輪圖示）

### 1.3 找到 Pages 設定

- 在左側選單向下滾動
- 找到「**Pages**」並點擊

### 1.4 配置 GitHub Pages

您會看到這個畫面：

```
GitHub Pages
────────────────────────────────────

Build and deployment

Source
[Deploy from a branch ▼]

Branch
[None ▼]  [/(root) ▼]  [Save]
```

**請這樣設定：**

1. **Source**（來源）
   - 選擇：「**Deploy from a branch**」

2. **Branch**（分支）
   - 第一個下拉選單：選擇「**main**」
   - 第二個下拉選單：選擇「**/ (root)**」

3. **點擊「Save」按鈕**

### 1.5 等待部署

點擊 Save 後：

```
✅ Your site is live at https://josiehsieh.github.io/staff-scheduling-system/
```

會出現這個訊息，但網站可能還需要 **2-3 分鐘**才會真正生效。

---

## 🌐 步驟 2：開啟網站

### 方法 A：直接在瀏覽器輸入網址（最簡單）

在瀏覽器網址列輸入：
```
https://josiehsieh.github.io/staff-scheduling-system/
```

按 Enter，就會開啟您的排班系統！

### 方法 B：從 GitHub 開啟

#### 選項 1：從 Settings 頁面
1. 在 Settings → Pages 頁面
2. 看到「Your site is live at...」
3. 點擊那個連結

#### 選項 2：從 Repository 首頁
1. 回到 Repository 首頁
2. 右側會看到「**Environments**」
3. 點擊「**github-pages**」
4. 點擊「**View deployment**」

#### 選項 3：從 About 區域
1. Repository 首頁右側的 About 區域
2. 點擊齒輪圖示編輯
3. 勾選「**Use your GitHub Pages website**」
4. 儲存後，網址會出現在右側

## ⚠️ 常見問題

### Q：點擊網址顯示 404

**A：網站還在部署中**
- 等待 2-5 分鐘
- 重新整理頁面（按 F5）
- 如果超過 10 分鐘還是 404，檢查 Pages 設定

### Q：在哪裡確認部署狀態？

**A：三個地方可以查看**
1. Settings → Pages（查看是否啟用）
2. Actions 頁籤（查看部署進度）
3. Environments → github-pages（查看部署歷史）

### Q：部署成功但頁面空白

**A：檢查瀏覽器 Console**
- 按 F12
- 查看 Console 的錯誤訊息
- 可能是檔案路徑問題

### Q：想要自訂網域名稱

**A：可以設定**
1. Settings → Pages
2. Custom domain 輸入您的網域
3. 需要在網域商設定 DNS
