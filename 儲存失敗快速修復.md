# ⚡ 儲存失敗快速修復

**問題：** 新增事業單位時顯示「儲存失敗」  
**閱讀時間：** 30秒  
**解決時間：** 1分鐘

---

## 🔥 立即解決（3步驟）

### 步驟1：打開控制台看錯誤

```
按 F12 → Console 標籤 → 點擊儲存 → 看錯誤訊息
```

### 步驟2：根據錯誤類型處理

**看到「403」或「permission」？**
```
→ 點擊「登出」→ 重新登入 → 授權權限 ✓
```

**看到「404」或「not found」？**
```
→ 到設定頁籤 → 檢查 Sheet ID → 重新填入 ✓
```

**看到「400」或「Unable to parse range」？**
```
→ 系統會自動建立工作表 → 重新儲存一次 ✓
```

**其他錯誤？**
```
→ Ctrl+Shift+R 強制重新整理 → 重新登入 ✓
```

### 步驟3：重新儲存

```
填寫資料 → 點擊儲存 → 成功！✅
```

---

## 📊 錯誤訊息對照表

| 控制台訊息 | 原因 | 解決方法 |
|-----------|------|---------|
| `status: 403` | 權限不足 | 重新登入 |
| `status: 404` | Sheet ID錯誤 | 檢查設定 |
| `status: 400` | 工作表不存在 | 重新儲存（自動建立） |
| `Unable to parse range` | 工作表不存在 | 重新儲存（自動建立） |
| `Network error` | 網路問題 | 檢查網路 |

## 🎯 最常見原因 TOP 3

### 1️⃣ 工作表不存在（最常見）

**症狀：**
```
❌ 儲存失敗
工作表格式錯誤！
```

**解決：**
```
系統現在會自動建立工作表！
→ 重新點擊「儲存」一次即可
```

### 2️⃣ 權限不足

**症狀：**
```
❌ 儲存失敗
權限不足！
```

**解決：**
```
登出 → 重新登入 → 授權 Sheets 權限
```

### 3️⃣ Sheet ID 錯誤

**症狀：**
```
❌ 儲存失敗
Sheet ID 錯誤！
```

**解決：**
```
複製 Google Sheets 網址中的 ID
→ 到設定頁籤貼上
```
